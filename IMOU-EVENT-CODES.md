# Imou Camera Event Codes Reference

## Imou Event Code Mapping

Based on real camera data, the Imou camera sends webhook events with numeric event codes.
The gateway automatically maps these codes to readable event types.

### Event Code Mapping

| Code | Event Type | Description |
|------|-----------|-------------|
| `0` | Unknown | Unknown event |
| `1` | Motion | Motion detection triggered |
| `2` | Person Detection | Person/human detected |
| `3` | Alarm | Alarm event triggered |
| `4` | Face Detection | Face detected |
| `5` | Vehicle Detection | Vehicle/car detected |
| `6` | Sound Detection | Abnormal sound detected |
| `7` | Tampering | Camera tampering detected |
| `8` | Line Crossing | Object crosses virtual line |
| `9` | Intrusion | Intrusion detection triggered |
| `10` | Parking | Parking/stationary object |

## Real Data Example

### Original Imou Webhook Data
```json
{
  "channel": 0,
  "event_type": "1",
  "description": "",
  "timestamp": "2026-07-04 14:31:18"
}
```

### Parsed by Gateway
```json
{
  "channel": 1,
  "type": "motion",
  "description": "Motion",
  "timestamp": "2026-07-04T14:31:18"
}
```

### Changes Made
- Channel: 0 → 1 (mapped to main channel)
- Event_type: "1" → "motion" (mapped using event code)
- Description: "" → "Motion" (auto-generated from event type)
- Timestamp: "2026-07-04 14:31:18" → "2026-07-04T14:31:18" (normalized to ISO format)

## Imou Camera Data Format

### Standard Format (Real Camera)
```
{
  "channel": 0,              // 0 = device-wide event
  "event_type": "1",         // Numeric event code
  "description": "",         // Often empty, auto-generated by parser
  "timestamp": "YYYY-MM-DD HH:MM:SS"  // Local time format
}
```

### Alternative Formats Supported
```
{
  "channel": 1,
  "type": "motion",          // String event type
  "description": "Motion",
  "timestamp": "2026-07-04T14:31:18"  // ISO format
}
```

## MQTT Publication

After parsing, events are published to:

### Motion Event (Code 1)
```
Topic: imou/camera/1/motion
Payload: ON

(After timeout, payload becomes: OFF)
```

### General Event
```
Topic: imou/event
Payload: {
  "channel": 1,
  "type": "motion",
  "description": "Motion",
  "timestamp": "2026-07-04T14:31:18",
  "device_id": "unknown"
}
```

## Channel Mapping

| Imou Channel | Gateway Channel | Notes |
|-------------|-----------------|-------|
| 0 | 1 | Device-wide events (main channel) |
| 1 | 1 | Primary camera channel |
| 2 | 2 | Secondary camera (if multi-channel) |
| N | N | Any numbered channel |

## Timestamp Formats Supported

The parser automatically normalizes timestamps from multiple formats:

| Format | Example | Result |
|--------|---------|--------|
| Imou Standard | `2026-07-04 14:31:18` | `2026-07-04T14:31:18` |
| ISO Format | `2026-07-04T14:31:18` | `2026-07-04T14:31:18` |
| ISO with Z | `2026-07-04T14:31:18Z` | `2026-07-04T14:31:18Z` |
| Alternative | `2026/07/04 14:31:18` | `2026-07-04T14:31:18` |

## Adding Custom Event Codes

To add support for additional Imou event codes:

1. Open `parser.py`
2. Locate `IMOU_EVENT_CODE_MAP` dictionary
3. Add new mapping:
   ```python
   IMOU_EVENT_CODE_MAP = {
       ...
       "11": "your_new_event",  # Add here
       ...
   }
   ```
4. Save and restart gateway

## Troubleshooting

### Issue: Events stored with numeric event_type (not mapped)
**Cause**: Event code not in mapping
**Solution**: Check event code, add to `IMOU_EVENT_CODE_MAP` in parser.py

### Issue: Events have empty description
**Cause**: Camera doesn't send description
**Solution**: Parser auto-generates from event type - this is normal

### Issue: Channel shows as 0
**Cause**: Device-wide event or multi-camera setup
**Solution**: Gateway automatically maps 0 → 1, check /events endpoint

### Issue: Timestamp format incorrect
**Cause**: Unsupported timestamp format from camera
**Solution**: Add format to `normalize_timestamp()` in parser.py
